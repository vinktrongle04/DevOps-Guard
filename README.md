## What is DevOps-Guard?

DevOps-Guard is a **zero-infrastructure, local-first DevSecOps platform** that transforms TRAE SOLO from a code editor into an **autonomous CI/CD governance brain**.

Instead of discovering security issues after they reach GitHub CI (too late), DevOps-Guard enforces 5 sequential Quality Gates **before the commit exists** — at the developer's workstation, in real time, without sending a single line of code to any external server.

> **The core insight:** Every other security tool (GitHub Advanced Security, Snyk, SonarQube, GitGuardian) operates *after* the code has been committed. Once a secret enters git history, it can never be truly erased. DevOps-Guard is the **prevention layer** — secrets never exist in commits in the first place.

## How TRAE SOLO Orchestrates the Pipeline

DevOps-Guard's unique advantage: **TRAE SOLO reads `project_rules.md` and `security_rules.md` as persistent brain files** and acts as the intelligence layer connecting all 5 gates.

```
Developer prompt: "Clean up my codebase before I commit."

TRAE reads:  project_rules.md → Pipeline Order 1→2→3→4→5
             security_rules.md → 28 rules + compliance mapping

TRAE executes:
  [Gate 1] Scans 28 rules → 8 CRITICAL found → replaces with env vars
  [Gate 2] Scans 16 packages → 4 unused → runs npm uninstall
  [Gate 3] Scans 4 components → migrates forwardRef + useContext
  [Gate 4] Documents changed components → appends to API_DOCUMENTATION.md
  [Gate 5] Reads git diff → generates Conventional Commit message

Result: 30 minutes of manual work → 42 seconds.
```

This is **Semantic Agent orchestration** — not regex matching. TRAE understands *why* a pattern exists and can fix it safely. Traditional scanners (Semgrep, SonarQube) detect problems. DevOps-Guard + TRAE **resolves** them.

## CI/CD Portability — SARIF + JSON Output

DevOps-Guard is not locked to GitHub Actions. It outputs industry-standard formats:

```bash
# Machine-readable JSON → Jenkins / GitLab CI / Azure DevOps / Splunk
npm run security:json > report.json

# SARIF 2.1.0 → GitHub Code Scanning / Azure DevOps Code Analysis
npm run security:sarif > results.sarif

# Combined report (Gate 1 + Gate 2) → Dashboard UI
npm run scan:export
```

| Splunk / BI | `npm run security:json` | JSON ingest pipeline |

## Scan History — Git-Native Telemetry

Every `npm run scan:export` appends a snapshot to `public/scan-history.json`. Since this file is committed to the repository, it provides **decentralized telemetry without a server**:

- Tech Lead runs `git pull` → receives updated policy rules AND latest metrics
- Compliance auditor runs `git clone` → gets full violation history instantly
- Dashboard renders trend sparkline from the local JSON — **zero cloud dependency**

```

76 ─
65 ─
57 ── May 30  (today: 36% improvement in 10 days) ●green
```

## Quick Start

```bash
# 1. Clone
git clone https://github.com/vinktrongle04/DevOps-Guard.git
cd DevOps-Guard

# 2. Install (Husky activates automatically)
npm install

# 3. Generate scan report
npm run scan:export

# 4. Open Dashboard
npm run dev
# → http://localhost:5173

# 5. Try a test commit (will be blocked)
echo 'const key = "AIzaSyFakeKey00000000000000000000000"' >> test.js
git add test.js && git commit -m "test"
# → ❌ COMMIT BLOCKED — [GOOG-001] CRITICAL
rm test.js
```
